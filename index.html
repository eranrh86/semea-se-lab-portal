What is Claude? — Core Concepts

Claude — The AI

Claude is an AI assistant built by Anthropic. Unlike a chatbot, Claude can take actions — read and write files, run shell commands, call APIs, browse the web, and connect to external services. In this lab we use it as a coding and infrastructure agent: you describe what you want in plain English, Claude figures out the commands and executes them.

🌐 claude.ai

The web interface. Good for conversations, document analysis, and one-off tasks. Supports MCP via integrations. Requires a claude.ai subscription.

💻 Claude Code (CLI)

The terminal agent used in this lab. Runs in your shell, reads your filesystem, executes commands, and operates autonomously. Installed via npm. claude --dangerously-skip-permissions skips confirmation prompts — essential for demos.

📱 Claude API

For building your own AI-powered applications. Not used in this lab directly, but the same models power Claude Code and claude.ai.

MCP — Model Context Protocol

MCP is the standard that lets Claude connect to external services. Think of MCP servers as plugins: each one gives Claude a set of tools it can call — query Datadog metrics, search Jira tickets, read Slack messages, list Google Drive files. You install them once and they're available in every session.

Tips & Best Practices

✅ Be Specific in Prompts

Include account name, timeframe, and desired format. "Summarise last 3 months of activity for CheckPoint" gets far better results than "what's happening with my accounts."

✅ Use CLAUDE.md for Persistent Context

Add ~/.claude/CLAUDE.md with your role, team, and common instructions. Claude reads it at the start of every session — you never repeat yourself.

⚠️ Limit Active MCP Tools

In Cursor, disable MCP tools you don't need for the current task. Each active tool consumes context window space and can slow responses. Connect Slack only when you need Slack.

⚠️ Avoid the GitHub MCP

It exposes too many tools and degrades model performance. Use the gh CLI instead — Claude runs it via shell and you get the same results with less noise.

🚫 Always Review Before Sending

AI-drafted emails, Jira updates, and Slack messages should always be reviewed before sending. Claude is a first-draft engine, not an autopilot. Especially for customer-facing content.

🐶 Use --dangerously-skip-permissions for Demos

Running cl (alias for claude --dangerously-skip-permissions) lets Claude execute commands without confirmation prompts — essential for live demos. Only use in your own terminal session.

Troubleshooting — Common Issues

⏳ Pod Stuck in Pending

Karpenter can't find a matching node or node isn't ready yet.

bash
kubectl describe pod <pod-name> -n <namespace>
kubectl get events -n <namespace> --sort-by='.lastTimestamp'
kubectl logs -n kube-system -l app.kubernetes.io/name=karpenter

Common causes: NodePool instance types not available in AZ, IAM permissions missing, resource requests exceed NodePool limits

🔌 kubectl: Can't Connect to Cluster

Credentials expired or kubeconfig not updated.

bash
source /tmp/aws_env.sh
aws sts get-caller-identity
aws eks update-kubeconfig --name semea-eks-<your-name> --region us-east-1

If tokens show expired: get new credentials from AWS Console → your account name → "Command line or programmatic access"

🔑 MCP Auth Fails / Disconnects

OAuth token expired or MCP server unreachable.

bash — Claude Code
claude mcp list          # see all registered MCPs
claude mcp remove slack  # remove and re-add if broken
claude mcp add --transport http --client-id 1601185624273.8899143856786 --callback-port 3118 slack https://mcp.slack.com/mcp

For Atlassian: sign in at mcp.atlassian.com and re-authorise. For Datadog: re-paste your App Key when Claude asks.

💸 AWS ExpiredTokenException

Session tokens expire every ~1 hour. Normal with SSO-based credentials.

paste into Claude
Credentials expired — update /tmp/aws_env.sh and continue:
export AWS_ACCESS_KEY_ID="<new-key>"
export AWS_SECRET_ACCESS_KEY="<new-secret>"
export AWS_SESSION_TOKEN="<new-token>"
🐶 Datadog Agent Not Sending Data

API Key wrong, agent not running, or network policy blocking egress.

bash
kubectl get pods -n datadog
kubectl logs -n datadog -l app=datadog-agent --tail=50
kubectl exec -n datadog <agent-pod> -- agent status

Check the API Key is correct in the secret: kubectl get secret datadog-secret -n datadog -o yaml

📦 ImagePullBackOff

Image name typo, private registry, or rate limit hit.

bash
kubectl describe pod <pod-name> -n <namespace> | grep -A5 Events
# Check the image name exactly — no trailing spaces, correct tag

For Docker Hub rate limits: add imagePullPolicy: IfNotPresent or use ECR public mirror
